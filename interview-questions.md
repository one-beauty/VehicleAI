## P0 级题目（高优先级 - 必须掌握）

### 1. 设计支持 10 万 QPS 的车载语音 API 网关

**问题**：如何设计一个支持 10 万 QPS 的高性能网关来处理车载语音请求？

**核心知识点**：
- 网关架构设计
- 异步非阻塞处理
- 虚拟线程应用
- 多级缓存策略
- 限流熔断

**关键技术栈**：
- Spring Cloud Gateway
- WebFlux
- Virtual Thread
- Caffeine + Redis
- Sentinel

**预期回答要点**：
1. 使用 Spring Cloud Gateway 基于 WebFlux 的异步非阻塞架构
2. 集成虚拟线程处理业务逻辑，单机支撑 1 万+ 并发连接
3. 构建二级缓存系统：Caffeine（本地）+ Redis（分布式）
4. 实现 Sentinel 限流规则，QPS 限制为 10000
5. 配置熔断降级策略，应对突发流量

**预计时间**：10-15 分钟

### 2. Spring Boot 服务如何实现优雅上下线？

**问题**：在 Kubernetes 环境中，Spring Boot 服务如何实现优雅上下线，避免请求丢失？

**核心知识点**：
- K8s 生命周期钩子
- preStop 钩子
- 连接管理
- 请求完成等待

**关键技术栈**：
- Kubernetes preStop
- Spring Boot Graceful Shutdown
- terminationGracePeriodSeconds

**预期回答要点**：
1. 配置 preStop 钩子，等待现有连接完成
2. 设置 terminationGracePeriodSeconds（如 30 秒）
3. 在 Spring Boot 中配置 server.shutdown=graceful
4. 设置 spring.lifecycle.timeout-per-shutdown-phase（如 30 秒）
5. 验证所有请求都被正确处理

**预计时间**：8-10 分钟

### 3. 虚拟线程 vs 传统线程池在语音流处理中的选择

**问题**：在处理大量并发语音流时，虚拟线程相比传统线程池有什么优势？

**核心知识点**：
- 虚拟线程原理
- 内存占用对比
- 并发能力对比
- 适用场景

**关键数据**：
- 传统线程：1MB 内存
- 虚拟线程：1KB 内存
- 虚拟线程支持 100 万+ 并发

**预期回答要点**：
1. 虚拟线程是轻量级线程，内存占用仅 1KB
2. 支持 100 万+ 并发连接，传统线程池只能支持 1000-10000
3. 虚拟线程由 JVM 调度，无需 OS 线程切换
4. 特别适合 I/O 密集型场景（如语音流处理）
5. 不适合 CPU 密集型任务

**预计时间**：10 分钟

### 6. 向量数据库选型：Milvus vs PGVector vs Elasticsearch

**问题**：在车载 AI 系统中，如何选择合适的向量数据库？

**核心知识点**：
- 性能对比
- 扩展性对比
- 成本对比
- 适用场景

**性能对比**：
- Milvus：< 100ms（十亿级数据）
- PGVector：200-500ms（百万级数据）
- Elasticsearch：150-300ms（百万级数据）

**预期回答要点**：
1. Milvus 性能最优，支持十亿级数据，延迟 < 100ms
2. PGVector 成本低，适合中小规模应用
3. Elasticsearch 功能全面，支持混合检索
4. 对于车载系统，推荐 Milvus（性能优先）
5. 可考虑 Milvus + Elasticsearch 混合方案

**预计时间**：10-12 分钟

### 2. 二叉树遍历（前序、中序、后序）

**问题**：实现二叉树的前序、中序、后序遍历，要求同时提供递归和迭代实现。

**核心知识点**：
- 递归遍历
- 迭代遍历（使用栈）
- 遍历顺序

**预期回答要点**：
1. 前序遍历：根 -> 左 -> 右
2. 中序遍历：左 -> 根 -> 右
3. 后序遍历：左 -> 右 -> 根
4. 递归实现简洁易懂
5. 迭代实现需要使用栈模拟递归

**预计时间**：8 分钟

### 3. Top K 问题（最小堆实现）

**问题**：给定一个数组，找出最大的 K 个元素，要求时间复杂度为 O(n log k)。

**核心知识点**：
- 最小堆
- 优先队列
- 堆排序

**预期回答要点**：
1. 使用最小堆维护 K 个最大元素
2. 遍历数组，对每个元素：
   - 如果堆大小 < K，直接加入
   - 如果元素 > 堆顶，删除堆顶并加入该元素
3. 时间复杂度：O(n log k)
4. 空间复杂度：O(k)

**预计时间**：10 分钟

## 追问示例

### 关于 RAG 系统的追问

**追问 1**：如何处理 RAG 系统中的冷启动问题？
**回答**：预加载热点数据到缓存，使用 Embedding 缓存避免重复计算，构建分层索引快速定位相关文档。

**追问 2**：Milvus 如何保证数据一致性？
**回答**：支持 ACID 事务，使用 WAL（Write-Ahead Logging），支持多副本和故障转移。

### 关于虚拟线程的追问

**追问 1**：虚拟线程在 CPU 密集型任务中表现如何？
**回答**：虚拟线程不适合 CPU 密集型任务，因为 JVM 调度器无法有效利用多核。应该使用传统线程池配合 ForkJoinPool。

**追问 2**：虚拟线程的上下文切换开销是多少？
**回答**：虚拟线程的上下文切换由 JVM 管理，开销远低于 OS 线程切换，大约是 OS 线程切换的 1/100。
